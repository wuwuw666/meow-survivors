# 塔改造系统 (Tower Mod System)

> **Status**: In Design
> **Author**: [user + agents]
> **Last Updated**: 2026-04-10
> **Implements Pillar**: 策略有深度 + 成长的爽感

---

## Overview

塔改造系统负责塔在**单局内**的战术变化。

它的定位不是传统塔升级树，而是通过精英敌人、Boss 奖励等战斗掉落，为塔提供本局临时改造件，让同一种塔在不同 run 中产生不同打法。

在当前 `C-Tangy` 成长结构中：

- 角色局内成长：由 `xp-system` 和角色升级系统负责
- 塔局内变化：由 `tower-mod-system` 负责
- 塔局外成长：由结算与局外成长系统负责

---

## Design Goal

塔改造系统要解决的问题是：

**在不把塔成长塞进 XP 升级池的前提下，让塔在本局里仍然有明显的 build 变化。**

玩家应该感受到：

- “这把我给这座塔装了不一样的东西”
- “同样是鱼干塔，这一局和上一局打法不一样”
- “防线不是只靠数值变强，而是在逐步长出花样”

---

## Core Loop

```text
击杀精英 / 完成 Boss 奖励
-> 获得塔改造候选
-> 选择 1 个改造件
-> 选择 1 座兼容塔进行装配
-> 该塔在本局中改变攻击或联动行为
```

## Drop Rules

### 规则 1：主要来源

MVP 主要来源：

- 精英敌人死亡：提供一次基础改造选择
- Boss 奖励：提供一次高品质改造选择

MVP 不建议：

- 普通敌人大量随机掉落
- 地面铺满拾取物

### 规则 2：掉落形式

MVP 推荐使用：

- 精英 / Boss 触发 `3 选 1` 改造面板
- 选择后再指定装配目标

不采用：

- 实体掉落后玩家跑去捡
- 自动装配

设计意图：

- 节奏更清楚
- 实现成本更低
- build 决策更有分量

## Mod Categories

### 1. 攻击形态改造

改变塔的攻击方式。

示例：

- 连射
- 分叉弹
- 扇形散射
- 剑雨 / 落点型攻击

### 2. 伤害属性改造

改变命中后的附带效果。

示例：

- 点燃
- 流血
- 冰缓
- 对残血增伤

### 3. 覆盖方式改造

改变塔对战场的覆盖方式。

示例：

- 范围扩大
- 小范围溅射
- 优先目标逻辑变化

### 4. 联动改造

改变塔与塔、塔与角色之间的关系。

示例：

- 受到辅助塔覆盖时额外触发
- 命中被英雄标记目标时追加效果
- 相邻同类塔存在时获得增强

## Equip Rules

### 规则 1：单局有效

改造件只在本局生效：

- 本局获得
- 本局生效
- 本局结束后清空

### 规则 2：必须绑定具体塔

改造件不是全局 buff，必须绑定到一座具体塔。

玩家需要做出：

- “这件改造给谁最值”
- “主力塔先吃，还是辅助塔先吃”

### 规则 3：每塔 1 槽

MVP 默认：

- 每座塔只有 `1` 个改造槽

后续可扩展为：

- 通过局外成长解锁第 `2` 槽

### 规则 4：冲突互斥

明显冲突的行为改造互斥。

示例：

- 连射 与 分叉鱼骨 互斥
- 缠绕毛线 与 散射毛线 互斥
- 浓缩猫薄荷 与 漫散猫薄荷 互斥

## MVP Mod List

### fish_shooter

- `连射鱼骨`：攻击改为三连发
- `分叉鱼骨`：命中后分裂小弹体
- `锯齿鱼骨`：对残血目标额外增伤

### yarn_launcher

- `缠绕毛线`：减速强化为强控
- `散射毛线`：攻击改为小范围散射
- `黏附毛线`：命中后使目标承受额外塔伤

### catnip_aura

- `浓缩猫薄荷`：范围缩小但 buff 更强
- `漫散猫薄荷`：范围扩大但强度下降
- `兴奋气味`：范围内英雄获得支援增益

## UI Flow

```text
精英死亡
-> 游戏暂停
-> 弹出 3 个改造候选
-> 玩家选择 1 个
-> 高亮兼容塔
-> 玩家点击装配目标
-> 反馈播放
-> 游戏恢复
```

UI 需要明确表达：

- 改造名称
- 一句话效果说明
- 适用塔类型
- 冲突或不可装配原因

## Dependencies

### 上游依赖

- 敌人系统：提供精英 / Boss 事件
- 战斗流程：控制暂停与恢复

### 下游依赖

- 塔系统：消费改造定义并改变塔行为
- UI 系统：显示候选与装配流程
- 结算系统：可记录本局关键改造组合

## Tuning Knobs

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `elite_mod_offer_count` | 3 | 精英改造候选数 |
| `boss_mod_offer_count` | 4 | Boss 改造候选数 |
| `mod_slot_count_per_tower` | 1 | 每塔改造槽数 |
| `pause_on_mod_offer` | true | 选择时是否暂停 |

## Edge Cases

| 编号 | 情况 | 处理方式 |
|---|---|---|
| EC-01 | 没有兼容塔 | 候选生成时尽量避免；如发生则取消本次装配并给补偿 |
| EC-02 | 塔已满槽 | MVP 默认不允许替换，避免生成该无效候选 |
| EC-03 | 玩家死亡与改造同时触发 | Game Over 优先 |
| EC-04 | 波次提示与改造面板冲突 | 改造面板优先 |

## Acceptance Criteria

| ID | 验证项 | Pass 标准 |
|---|---|---|
| AC-TM-01 | 精英触发改造选择 | 精英死亡后弹出改造候选 |
| AC-TM-02 | 玩家能选择改造 | 候选可被正确选择 |
| AC-TM-03 | 玩家能指定装配塔 | 兼容塔可高亮并可装配 |
| AC-TM-04 | 行为变化可见 | 装配后该塔行为立即发生明显变化 |
| AC-TM-05 | 单局清空 | 本局结束后改造件不跨局保留 |
