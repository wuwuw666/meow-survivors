# 塔局外成长系统 (Tower Meta Progression System)

> **Status**: In Design
> **Author**: [user + agents]
> **Last Updated**: 2026-04-15
> **Implements Pillar**: 成长的爽感 + 策略有深度

---

## Overview

塔局外成长系统负责管理塔在**多局之间**的长期成长。

在当前 `C-Tangy` 成长结构中：

- 角色局内成长：由 XP 与角色升级系统负责
- 塔局内变化：由塔改造系统负责
- 塔局外成长：由本系统负责

这个系统消耗结算系统发放的 `tower_scrap` 或同类资源，用来推进塔体系的长期建设。

它的目标不是让玩家靠账号强度碾压单局，而是让玩家逐步获得：

- 更多塔选项
- 更稳定的基础塔强度
- 更丰富的塔分支方向
- 更高的塔 build 上限

---

## Design Goal

塔局外成长系统要实现三件事：

1. 让单局结果有长期价值
2. 让玩家觉得自己的塔体系在持续成熟
3. 不让长期成长压过单局布阵与局内改造的决策价值

也就是说，这个系统应该提供**长期方向感**，但不能替代：

- 塔位决策
- 塔局内改造选择
- 角色局内支援 build

---

## Core Loop

```text
完成一局
-> 结算获得 tower_scrap
-> 进入局外成长界面
-> 选择解锁或升级某个塔成长节点
-> 下一局带着新的塔体系进入战斗
```

---

## Progression Scope

MVP 建议局外成长只覆盖以下三类内容：

### 1. 塔解锁

作用：

- 解锁新的塔类型
- 扩大玩家的塔阵构筑空间

示例：

- 解锁新的输出塔
- 解锁新的辅助塔

### 2. 基础塔强化

作用：

- 轻量提升塔的基础属性

示例：

- 基础伤害小幅提高
- 基础范围小幅提高
- 部署成本小幅优化

约束：

- 增幅必须轻量
- 不能让局外成长直接压平局内难度

### 3. 塔成长能力解锁

作用：

- 解锁未来局内 build 的额外空间

示例：

- 解锁第 2 个改造槽（v1.0 后）
- 解锁某塔的进阶分支
- 解锁额外改造件候选池

---

## MVP Recommendation

MVP 第一阶段建议只做：

- 新塔解锁
- 小幅基础强化

MVP 暂不建议一开始就做：

- 大型科技树
- 多层分支互斥网状结构
- 复杂槽位系统
- 强 Meta 被动连锁

设计意图：

- 先验证短局和成长闭环
- 再决定长期成长深度

---

## Example Node Categories

### Offensive Tower Nodes

- 输出塔基础伤害 +5%
- 输出塔基础范围 +5%
- 某输出塔解锁

### Effect Output Tower Nodes

- 效果输出塔减速持续时间 +10%
- 效果输出塔基础范围 +5%
- 某效果输出塔解锁

### Support Tower Nodes

- 辅助塔 aura 范围 +5%
- 辅助塔增益强度 +5%
- 某辅助塔解锁

## MVP Tower Progression Layout

MVP 建议采用一套非常轻量、非常直观的局外成长布局：

- 玩家开局默认拥有 `fish_shooter`
- 玩家通过结算获得 `tower_scrap`
- 玩家优先在 3 条简单成长线上做选择：
  - 解锁新塔
  - 轻量基础强化
  - 为后续 run 打开更多战术方向

MVP 不建议把界面做成复杂科技树，更适合使用：

- 左到右的简单节点链
- 按塔类型分组的分栏结构
- 清晰前置关系，不做复杂网状互斥

---

## MVP Starter State

MVP 推荐默认初始状态如下：

| 项目 | 初始状态 | 设计意图 |
|---|---|---|
| `fish_shooter` | 已解锁 | 给玩家稳定的基础主力塔 |
| `yarn_launcher` | 未解锁 | 作为第一批中期战术扩展 |
| `catnip_aura` | 未解锁 | 作为辅助体系的初始里程碑 |
| 输出塔基础强化 | 未购买 | 给早期资源一个稳定去处 |
| 辅助塔基础强化 | 未购买 | 让玩家逐步理解体系差异 |

设计意图：

- 第一批资源花费要有明确方向
- 玩家第一轮选择就能感受到“我是先拓宽，还是先稳一点”
- 初始状态不应过空，否则结算奖励缺乏意义

## Purchase Priority Guidance

为了让早期局外成长更好理解，MVP 推荐给玩家隐含形成这三种购买路线：

### 路线 1：先稳主力输出

适合第一次接触系统的玩家：

- `tmp_fish_damage_1`
- `tmp_fish_range_1`
- `tmp_fish_damage_2`

体验目标：

- 下一局更容易感受到基础防线更稳

### 路线 2：先解锁第二种输出功能

适合想要更早体验控制与效果分支的玩家：

- `tmp_unlock_yarn`
- `tmp_yarn_slow_1`
- `tmp_yarn_range_1`

体验目标：

- 下一局能明显体验“效果输出塔”加入后的防线变化

### 路线 3：先做辅助塔体系

适合想要更早玩联动和站位价值的玩家：

- `tmp_unlock_catnip`
- `tmp_catnip_aura_1`
- `tmp_catnip_power_1`

体验目标：

- 让玩家更快理解“辅助塔放在哪”这件事本身就是策略

## Node Count Budget

MVP 第一阶段建议把局外成长节点总量控制在：

- 10 到 14 个可购买节点

推荐当前目标：

- 1 个默认起始节点
- 4 个第一层节点
- 5 个第二层节点
- 3 个第三层节点

设计意图：

- 足够形成早期长期推进感
- 不会让 UI 和数值维护过早膨胀
- 也不会在短局 MVP 阶段引入重 Meta 压力

---

## Unlock And Purchase Rules

### 规则 1：单节点购买要立即可理解

玩家购买节点后，应该能直接回答：

- “我下一局多了什么”
- “我下一局哪类塔更稳了”
- “我下一局可以试什么新组合”

### 规则 2：解锁优先于堆数值

MVP 前期节点分布要优先鼓励：

- 解锁新塔
- 解锁新战术方向

而不是让玩家一开始就只会把资源全部砸进单一伤害加成。

### 规则 3：基础强化应有软上限

MVP 建议单塔基础强化上限控制在：

- 伤害类总加成不超过 +10% 到 +15%
- 范围类总加成不超过 +10% 到 +15%
- 辅助类总加成不超过 +10% 到 +15%

设计意图：

- 长期成长是轻推
- 单局表现仍主要由放塔、改造和角色支援决定

---

## Resource Model

本系统默认消耗：

- `tower_scrap`

来源：

- `settlement-system`

设计意图：

- 统一长期成长货币
- 避免一开始就引入过多资源种类

后续可扩展：

- 稀有塔核心
- Boss 结晶
- 分支专属材料

但 MVP 暂不需要。

---

## Upgrade Rules

### 规则 1：局外成长应偏“拓宽与轻推”

局外成长优先提供：

- 更多选择
- 更完整体系
- 更稳定基础

而不是直接提供：

- 爆炸式伤害增长
- 绝对安全的早期局面
- 无需布阵的账号碾压

### 规则 2：单点升级要有明确价值

每次局外成长最好回答一个清晰问题：

- “我解锁了什么新战术”
- “我哪类塔更稳定了”
- “我下一局能尝试什么新方向”

### 规则 3：不替代局内 build

局外成长不能让玩家觉得：

- “我这局赢只是因为账号养成高”

理想状态是：

- 账号成长决定可用空间
- 单局表现决定这把怎么赢

## UI Structure

MVP 局外成长界面建议分为三栏：

### 1. 当前资源

显示：

- 当前 `tower_scrap`

### 2. 塔类别

按类型分栏：

- 输出塔
- 效果输出塔
- 辅助塔

### 3. 可升级节点

每个节点显示：

- 名称
- 效果
- 花费
- 当前状态（已解锁 / 可购买 / 未解锁）

## Dependencies

### 上游依赖

- `settlement-system`：提供结算资源
- 未来 `save-system`：保存玩家长期进度

### 下游依赖

- `tower-system`：读取已解锁塔与基础强化
- `tower-mod-system`：读取是否解锁额外改造容量或改造池
- UI 系统：展示局外成长界面

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `base_unlock_cost` | 20 | 新塔基础解锁成本 |
| `base_stat_upgrade_cost` | 10 | 小幅属性强化成本 |

| `max_mvp_stat_bonus` | 0.15 | MVP 期单项基础加成上限 |

## Edge Cases

| 编号 | 情况 | 处理方式 |
|---|---|---|
| EC-01 | 玩家资源不足 | 节点显示但不可购买 |
| EC-02 | 已满级节点再次点击 | 不消费资源并提示已满级 |
| EC-03 | 新塔未解锁但其分支节点可见 | 置灰并显示前置条件 |
| EC-04 | 存档缺失或损坏 | 回退到默认初始塔体系 |

## Acceptance Criteria

| ID | 验证项 | Pass 标准 |
|---|---|---|
| AC-TMP-01 | 结算资源接入 | 结算获得的 `tower_scrap` 可进入局外成长系统 |
| AC-TMP-02 | 新塔解锁 | 玩家可用资源解锁新塔类型 |
| AC-TMP-03 | 基础强化生效 | 已购买的基础强化可在下一局被塔系统读取 |
| AC-TMP-04 | 边界清晰 | 局外成长不会直接替代局内改造和角色升级 |
| AC-TMP-05 | 长期推进感 | 玩家能理解“每局都在推进塔体系” |
