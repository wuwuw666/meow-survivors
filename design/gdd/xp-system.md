# 经验系统 (XP System)

> **Status**: In Design
> **Author**: [user + agents]
> **Last Updated**: 2026-04-10
> **Implements Pillar**: 成长的爽感

---

## Overview

经验系统负责管理角色在单局中的经验获取、等级提升和升级触发。

在当前 `C-Tangy` 成长结构中，经验系统只服务于**角色局内成长**，不再直接驱动塔的成长。塔的局内变化由 `tower-mod-system` 负责，塔的长期成长由局外成长系统负责。

经验系统的核心职责是：

- 记录本局角色经验
- 计算升级阈值
- 在经验满足条件时触发升级事件
- 向 UI 提供经验条与等级信息

经验系统**不负责**：

- 决定升级候选内容
- 决定塔的局内改造
- 决定局外塔成长资源

---

## Player Fantasy

玩家应该感受到：

- 击杀敌人后，角色在稳定成长
- 经验条推进清晰，升级节奏可预期
- 每次升级都意味着一次新的角色支援选择

玩家不应该感受到：

- 角色经验同时在偷偷强化塔，导致成长来源混乱
- 经验满了却没有明确升级反馈
- XP 系统和塔改造、结算奖励混在一起

---

## Core Loop

```text
击杀敌人
-> 获得 XP
-> XP 条增长
-> 达到升级阈值
-> 触发 level_up_requested
-> 升级选择系统弹出角色升级面板
-> 玩家选择 1 个角色升级
-> XP 系统确认本次升级完成
```

---

## Detailed Rules

### 规则 1：XP 只用于角色升级

XP 系统只驱动角色成长，不直接提供以下收益：

- 塔伤害提升
- 塔攻速提升
- 塔范围提升
- 塔部署资源
- 塔局内改造件

这些内容分别由其他系统负责：

- 角色升级内容：`upgrade-selection-system`
- 塔局内变化：`tower-mod-system`
- 塔长期成长：结算与局外成长系统

### 规则 2：经验来源

MVP 默认经验来源：

- 普通敌人击杀
- 精英敌人击杀
- Boss 击杀

经验来源建议示例：

| 敌人类型 | XP |
|---|---:|
| 普通敌人 | 3 |
| 强化敌人 | 5 |
| 精英敌人 | 12 |
| Boss | 40 |

MVP 不额外加入复杂来源，例如：

- 塔改造拾取转 XP
- 结算奖励转 XP
- 塔击杀额外倍率加成

### 规则 3：升级阈值

经验需求采用稳定上升曲线，保证前期较快成型，中后期升级节奏收紧。

建议公式：

```text
xp_needed(level) = ceil(base_xp * (1 + (level - 1) * growth_rate))
base_xp = 10
growth_rate = 0.3
```

示例：

| 当前等级 | 升到下一级所需 XP |
|---|---:|
| 1 -> 2 | 10 |
| 2 -> 3 | 13 |
| 3 -> 4 | 16 |
| 4 -> 5 | 19 |
| 5 -> 6 | 22 |

### 规则 4：升级触发

```text
获得 XP
-> current_xp 增加
-> 如果 current_xp >= xp_needed
-> 触发一次角色升级请求
```

经验系统应支持一次获得大量经验时的连续升级检查，但具体面板弹出由升级选择系统顺序处理。

### 规则 5：升级确认

经验系统在收到角色升级选择完成的确认后：

- 扣除本级所需经验
- 提升角色等级
- 刷新下一等级所需经验
- 若剩余经验仍足够，则允许再次触发升级请求

这样可以避免：

- 经验条超出后丢失经验
- 连续升级时状态错乱

## Core Interface

- `add_xp(amount: int, source: String = "enemy_kill") -> void`
- `get_level() -> int`
- `get_current_xp() -> int`
- `get_xp_needed() -> int`
- `confirm_level_up() -> void`
- `signal xp_changed(current: int, needed: int)`
- `signal level_up_requested(new_level: int)`

---

## Dependencies

### 上游依赖

- 敌人系统：提供击杀事件与经验值
- 战斗流程：保证暂停时经验拾取不会异常结算

### 下游依赖

- 升级选择系统：消费 `level_up_requested`
- UI 系统：消费 `xp_changed`
- 结算系统：读取本局总获得 XP 作为统计项

---

## Tuning Knobs

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `base_xp` | 10 | 1 级升 2 级基础需求 |
| `growth_rate` | 0.3 | 每级需求增长率 |
| `normal_enemy_xp` | 3 | 普通敌人经验 |
| `elite_enemy_xp` | 12 | 精英敌人经验 |
| `boss_xp` | 40 | Boss 经验 |

---

## Edge Cases

| 编号 | 情况 | 处理方式 |
|---|---|---|
| EC-01 | 一次获得大量 XP 跨多级 | 顺序触发升级请求，不丢失剩余 XP |
| EC-02 | 升级面板期间再次满足升级条件 | 保留剩余 XP，待本次升级完成后继续检查 |
| EC-03 | 玩家死亡时经验未结算 | 经验停止增长，不再触发升级 |
| EC-04 | 波次结束与升级同时发生 | 升级面板优先，波次提示后延 |

## Acceptance Criteria

| ID | 验证项 | Pass 标准 |
|---|---|---|
| AC-XP-01 | 击杀获得经验 | 击杀敌人后 XP 正确增加 |
| AC-XP-02 | 经验条同步 | UI 能正确显示当前 XP 和需求 |
| AC-XP-03 | 升级触发 | XP 达标时发出 `level_up_requested` |
| AC-XP-04 | 连续升级 | 大额 XP 不丢失，并可连续升级 |
| AC-XP-05 | 角色专属成长 | XP 触发的升级不直接强化塔 |
