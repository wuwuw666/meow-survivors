---

## 1. Overview

地图系统是《喵族幸存者》的 Foundation 层系统，负责定义游戏场地的物理边界、可行走区域、塔位（Tower Slots）坐标与占用状态，以及刷怪点（Spawn Points）位置。它是纯数据提供者：不主动驱动任何逻辑，只向移动系统、塔位放置系统、敌人生成系统和难度曲线系统提供地图静态或只读数据。MVP 只需一张地图——"猫粮仓库"（Cat Food Warehouse），为整条核心循环的运行提供空间基础。

---

## 2. Player Fantasy

玩家踏入猫粮仓库的第一眼，应该感受到：这是一个 **"我能看懂、我能用上、我有地方可守"** 的空间。

- **看得懂布局**：塔位标记清晰，刷怪方向可以预判，玩家不需要读说明书就能直觉地找到"好站位"。
- **感受到空间的节奏**：狭窄走廊逼出走位压力，开阔区域给出喘息空间，地形本身就在讲述"此处危险，彼处安全"的故事。
- **塔位是策略锚点**：每个塔位不是随机摆放的圆圈，而是精心设计的"我想在这里放一座塔"的冲动点——走廊入口、路口节点、视野交汇处。
- **可爱且有代入感**：猫粮仓库主题带来温暖的空间感——木箱、麻袋、猫爪印，这不是冰冷的战场，是猫咪守护家园的舞台。

**玩家不应感受到**：
- 地图边界像一堵突然出现的隐形墙，打断走位节奏。
- 塔位被放置在对防守没有意义的角落。
- 刷怪点分布让某一侧完全无法防守。

## 3. Detailed Design

### 3.1 Core Rules

#### 规则 1：物理尺寸与边界

- 地图为矩形，MVP 固定尺寸：**1920 × 1088 像素**（即 60 × 34 个 32px 图格）。
- 边界为硬边界，玩家和敌人均不可穿越。
- 边界判定由移动系统执行，地图系统只提供 `map_bounds: Rect2` 数据。

#### 规则 2：可行走区域（Walkable Area）

- 地图分为"可行走"和"不可行走"两种图格。
- 不可行走区域包括：仓库墙体、超大木箱堆、地图装饰障碍物。
- 可行走区域约占地图总面积的 **70%**，保证玩家有足够走位空间。
- 可行走性以 `TileMap` 层表达，移动系统通过查询 `is_tile_walkable(tile_pos: Vector2i) -> bool` 判断。

#### 规则 3：塔位（Tower Slots）

- MVP 地图固定放置 **6 个塔位**。
- 每个塔位有三种状态：**空置（Empty）/ 已占用（Occupied）/ 锁定（Locked）**。
- MVP 中所有塔位初始为 **空置**，不使用锁定状态（锁定为 v1.0 功能预留）。
- 塔位为特定的地图坐标点，不随机生成。
- 塔位位置设计原则：
  - 至少 2 个位于走廊入口或路口，覆盖主要敌人路径。
  - 至少 2 个位于地图中段，能覆盖多个方向。
  - 至多 2 个位于地图深处，作为辅助支撑点。

**MVP 塔位布局（猫粮仓库）**：

| 塔位 ID | 位置描述         | 像素坐标（近似） | 设计意图           |

| TS-02   | 东北路口节点     | (1440, 224)     | 覆盖东、北两路     |
| TS-03   | 地图中心左侧     | (640, 544)      | 中场辅助输出       |

| TS-06   | 东南角战略点     | (1440, 864)     | 覆盖东、南两路     |

#### 规则 4：刷怪点（Spawn Points）

- MVP 地图固定 **8 个刷怪点**，分布于地图四条边沿。
- 刷怪点不在地图内部，而是在边界外侧，敌人生成后从该点向场内移动。
- 刷怪点坐标由地图系统提供，敌人生成系统选择使用哪些刷怪点。
- 设计目标：确保没有任何单一方向被彻底忽视，也没有任何方向压力过于集中。

#### 规则 5：地图压力修正（map_pressure_modifier）

- 地图系统向难度曲线系统提供 `map_pressure_modifier: float`，表示当前地图对玩家施加的额外空间压力。
- 计算基于地图面积和可行走区域比例（见第 4 节公式）。
- MVP 猫粮仓库为中等尺寸地图，`map_pressure_modifier` 预计值为 **1.0**（无额外修正）。
- 难度曲线系统将此修正作为全局系数，按软依赖使用（即使地图系统未返回值，难度曲线默认使用 1.0）。

#### 规则 6：地图只读原则

- 地图系统不修改任何运行时状态，**唯一例外**是塔位的占用状态。
- 塔位状态由塔位放置系统通过 `set_slot_occupied()` / `set_slot_empty()` 调用更新。
- 地图系统不主动发射信号驱动任何行为，只提供查询接口。

### 3.2 States and Transitions

地图系统本身极简，运行时只有两个状态：

| 状态           | 描述                         | 进入条件               | 持续期间行为                   |
|--------------|------------------------------|----------------------|------------------------------|
| **Unloaded** | 地图数据未加载                 | 游戏启动初期           | 拒绝所有查询请求，返回空/错误值   |
| **Active**   | 地图已加载，正在运行           | 场景加载完成后           | 提供全部查询接口，维护塔位状态    |

**状态转换**：

| 当前状态   | 触发事件               | 目标状态   |
|------------|----------------------|------------|
| Unloaded   | 地图场景加载完成       | Active     |
| Active     | 游戏结束/场景切换      | Unloaded   |

塔位状态机（每个塔位独立）：

| 当前状态   | 触发事件                           | 目标状态   |
|------------|----------------------------------|------------|
| Empty      | 塔位放置系统调用 `set_slot_occupied` | Occupied   |
| Occupied   | 塔位放置系统调用 `set_slot_empty`   | Empty      |
| Locked     | (MVP 不使用此状态)                 | —          |

### 3.3 Interactions with Other Systems

| 系统               | 交互方向                  | 接口                                          | 说明                                               |
|------------------|--------------------------|-----------------------------------------------|----------------------------------------------------|
| **移动系统**       | 地图 → 移动               | `map_bounds: Rect2`, `is_tile_walkable()`     | 移动系统用边界和可行走区域约束玩家及敌人移动              |
| **塔位放置系统**   | 双向                      | `get_all_slots()`, `set_slot_occupied()`, `set_slot_empty()` | 塔位放置系统读取塔位状态、更新占用             |
| **敌人生成系统**   | 地图 → 敌人生成            | `get_spawn_points() -> Array[Vector2]`        | 敌人生成系统选择刷怪点并在该坐标生成敌人               |
| **难度曲线系统**   | 地图 → 难度曲线（软依赖）  | `map_pressure_modifier: float`                | 难度曲线按修正值全局调整波次压力；地图未提供时默认 1.0   |

**WaveConfig 与地图系统的关系**：

难度曲线系统定义的 `WaveConfig` 使用刷怪点数据时，由敌人生成系统负责调用 `get_spawn_points()`，地图系统不直接参与 `WaveConfig` 的构建。`map_pressure_modifier` 在 `WaveConfig` 生成时由难度曲线系统读取并注入倍率计算。

参考 `WaveConfig` 结构（定义于难度曲线系统）：

```gdscript
class_name WaveConfig

var wave_number: int
var duration_sec: float
var spawn_budget: int
var spawn_interval_sec: float
var enemy_mix: Dictionary
var hp_multiplier: float
var damage_multiplier: float
var speed_multiplier: float
var elite_chance: float
var is_milestone: bool
var threat_level: int
var upgrade_pause_sec: float
# 注意：map_pressure_modifier 由难度曲线系统在构建 WaveConfig 时注入，
# 不作为 WaveConfig 的独立字段暴露给其他系统
```

## 4. Formulas

### 4.1 变量定义

| 变量                     | 含义                                   | 类型    | MVP 值 / 范围            |
|------------------------|--------------------------------------|---------|--------------------------|
| `MAP_WIDTH_PX`         | 地图像素宽度                           | int     | 1920                     |
| `MAP_HEIGHT_PX`        | 地图像素高度                           | int     | 1088                     |
| `TILE_SIZE`            | 单个图格像素尺寸                       | int     | 32                       |
| `map_area`             | 地图总图格面积（格数）                  | int     | `(1920/32) × (1088/32) = 60 × 34 = 2040` |
| `walkable_area`        | 可行走图格数量                         | int     | 约 1428（占 70%）         |
| `walkable_ratio`       | 可行走区域比率                         | float   | `walkable_area / map_area`，目标 0.65-0.75 |
| `ref_area`             | 参考标准地图面积（用于压力修正基准）     | int     | 2040（MVP 地图自身）      |
| `ref_walkable_ratio`   | 参考标准可行走比率                     | float   | 0.70                     |
| `map_pressure_modifier`| 地图压力修正系数                       | float   | 计算值，预期为 1.0        |
| `num_tower_slots`      | 塔位总数                               | int     | 6                        |
| `num_spawn_points`     | 刷怪点总数                             | int     | 8                        |

### 4.2 地图压力修正公式

```text
area_factor = ref_area / map_area
walkable_factor = ref_walkable_ratio / walkable_ratio

map_pressure_modifier = clamp(area_factor * walkable_factor, 0.70, 1.50)
```

**公式含义**：
- `area_factor`：地图面积越小，玩家回旋空间越少，压力越高。
- `walkable_factor`：可行走区域越窄（比率越低），走位约束越强，压力越高。
- `clamp` 防止极端地图配置导致难度曲线数值失控。

**MVP 示例计算（猫粮仓库）**：

```text
area_factor = 2040 / 2040 = 1.00
walkable_factor = 0.70 / 0.70 = 1.00
map_pressure_modifier = clamp(1.00 × 1.00, 0.70, 1.50) = 1.00
```

MVP 地图为基准地图，修正系数为 **1.0**，不对难度曲线产生额外影响。

**假设性示例（小型地图，v1.0 参考）**：

```text
# 假设小地图：1280 × 704 px，可行走比率 0.55（狭窄走廊地图）
map_area_small = (1280/32) × (704/32) = 40 × 22 = 880
area_factor = 2040 / 880 ≈ 2.32  → clamp 后上限为 1.50
walkable_factor = 0.70 / 0.55 ≈ 1.27
map_pressure_modifier = clamp(2.32 × 1.27, 0.70, 1.50) = clamp(2.95, 0.70, 1.50) = 1.50
```

小型狭窄地图触发最大压力修正，难度曲线全局系数 ×1.50。

### 4.3 塔位覆盖半径（参考值，由防御塔系统使用）

地图系统不计算覆盖，但记录塔位坐标精度要求：

```text
塔位坐标对齐到 32px 图格中心：
slot_world_pos = Vector2(tile_x * TILE_SIZE + TILE_SIZE / 2, tile_y * TILE_SIZE + TILE_SIZE / 2)
```

示例（TS-01，图格坐标 (15, 5)）：

```text
slot_world_pos = Vector2(15 * 32 + 16, 5 * 32 + 16) = Vector2(496, 176)
```

坐标以 Godot 世界坐标系表达，原点在地图左上角。

## 5. Edge Cases

| 边界情况                                           | 处理方式                                                                                                       |
|--------------------------------------------------|--------------------------------------------------------------------------------------------------------------|
| 玩家移动至地图边界                                 | 移动系统读取 `map_bounds` 并在边界处阻止移动；地图系统不参与，仅提供 `Rect2` 数据                                |
| 敌人生成时刷怪点被其他敌人堵塞                     | 敌人生成系统负责处理拥塞（换点或延迟生成），地图系统始终返回固定刷怪点坐标，不感知场上实体状态                      |
| 所有 6 个塔位均已被占用                            | 地图系统返回空数组给 `get_empty_slots()`；塔位放置系统负责告知玩家无空余塔位，地图系统不发出提示信号             |
| 塔位放置系统尝试占用已被占用的塔位                 | `set_slot_occupied()` 返回 `false`，并在 debug 模式下打印警告；塔位状态保持不变，不抛出异常                      |
| 塔位放置系统尝试清空空置的塔位                     | `set_slot_empty()` 返回 `false`，debug 模式打印警告；不影响运行                                                 |
| `is_tile_walkable()` 被传入越界坐标               | 返回 `false`（视为不可行走），不抛出异常；由调用方负责坐标合法性检查                                            |
| 地图系统在 Unloaded 状态下被查询                  | 所有查询接口返回安全的默认值（空数组、零向量、`false`），并在 debug 模式打印错误；不崩溃                          |
| `map_pressure_modifier` 计算结果超出 `[0.70, 1.50]` | 强制 clamp，记录设计警告（说明地图尺寸或通道率偏离设计预期），不中断运行                                        |
| 刷怪点坐标与塔位坐标完全重叠                       | 属于关卡设计错误，在地图资源导入时（`_ready()` 阶段）校验并打印警告，但不阻止运行；关卡设计师需手动修正           |
| 障碍物阻断所有刷怪路径（敌人无法到达玩家）         | 属于关卡设计错误，地图系统不负责路径检测；由敌人 AI 移动系统发现并上报；地图系统提供障碍物图格数据供调试使用      |

## 6. Dependencies

### 6.1 上游依赖

地图系统是 Foundation 层系统，**无运行时上游依赖**。

| 依赖项              | 类型     | 用途                                                         |
|-------------------|----------|--------------------------------------------------------------|
| 游戏概念文档        | 设计依赖 | 确定 MVP 为 1 张地图（猫粮仓库），规定地图主题和场景内容      |
| 系统索引            | 设计依赖 | 确认地图系统在 Foundation 层，无循环依赖                      |
| Godot TileMap API  | 技术依赖 | 地图数据使用 Godot 4.6 TileMap 节点存储可行走性和图格信息    |

### 6.2 下游依赖

| 系统               | 依赖类型 | 使用接口                                                      | 说明                                        |
|------------------|----------|-------------------------------------------------------------|---------------------------------------------|
| **移动系统**       | 硬依赖   | `map_bounds`, `is_tile_walkable()`                          | 玩家和敌人的移动边界与障碍物判断              |
| **塔位放置系统**   | 硬依赖   | `get_all_slots()`, `get_empty_slots()`, `set_slot_occupied()`, `set_slot_empty()` | 查询可放置位置、更新占用状态 |
| **敌人生成系统**   | 硬依赖   | `get_spawn_points()`                                        | 获取刷怪点坐标列表，由生成系统选择           |
| **难度曲线系统**   | 软依赖   | `map_pressure_modifier`                                     | 全局难度修正；未接入时默认值 1.0             |

### 6.3 接口定义

```gdscript
class_name MapSystem
extends Node

## 地图物理边界（世界坐标系，左上角为原点）
var map_bounds: Rect2

## 返回地图压力修正系数，供难度曲线系统使用
## 小地图或可行走区域窄的地图返回 > 1.0，宽敞地图返回接近 1.0
func get_map_pressure_modifier() -> float

## 查询指定图格坐标是否可行走
## tile_pos: TileMap 图格坐标（非世界坐标）
## 越界坐标返回 false
func is_tile_walkable(tile_pos: Vector2i) -> bool

## 返回所有刷怪点的世界坐标数组（固定8个，不含场上实体状态）
func get_spawn_points() -> Array[Vector2]

## 返回所有塔位数据（含状态）
func get_all_slots() -> Array[TowerSlotData]

## 返回当前状态为 Empty 的塔位数据
func get_empty_slots() -> Array[TowerSlotData]

## 返回当前状态为 Occupied 的塔位数据
func get_occupied_slots() -> Array[TowerSlotData]

## 将指定塔位标记为已占用
## 返回 true 表示成功，false 表示该塔位已被占用或不存在
func set_slot_occupied(slot_id: StringName) -> bool

## 将指定塔位标记为空置
## 返回 true 表示成功，false 表示该塔位已为空置或不存在
func set_slot_empty(slot_id: StringName) -> bool

## 根据 ID 获取单个塔位数据
## 不存在时返回 null
func get_slot_by_id(slot_id: StringName) -> TowerSlotData


## 塔位数据结构
class_name TowerSlotData

enum SlotState { EMPTY, OCCUPIED, LOCKED }

var slot_id: StringName          ## 唯一标识符，如 &"TS-01"
var world_pos: Vector2           ## 世界坐标（图格中心）
var tile_pos: Vector2i           ## TileMap 图格坐标
var state: SlotState             ## 当前状态
var occupied_by: StringName      ## 若为 OCCUPIED，记录塔的类型 ID；否则为空
```

## 7. Tuning Knobs

| 参数名                    | 类型    | 默认值   | 安全范围          | 影响                                                                    |
|-------------------------|---------|---------|-----------------|-------------------------------------------------------------------------|
| `MAP_WIDTH_PX`          | int     | 1920    | 1280-2560       | 地图宽度；超过 2560 会拉长走位距离，影响战斗节奏                          |
| `MAP_HEIGHT_PX`         | int     | 1088    | 768-1536        | 地图高度；低于 768 会使战场过于狭窄，高于 1536 则过于空旷                 |
| `num_tower_slots`       | int     | 6       | 4-8             | 塔位数量；低于 4 时策略选择过少，高于 8 时塔防比重过大，影响幸存者类手感   |
| `num_spawn_points`      | int     | 8       | 4-12            | 刷怪点数量；低于 4 时方向压力集中，失去包围感；高于 12 时刷怪点意义分散    |
| `walkable_ratio_target` | float   | 0.70    | 0.55-0.80       | 可行走区域目标比率；低于 0.55 时走位受限过度，高于 0.80 时缺乏障碍物策略感 |
| `pressure_modifier_min` | float   | 0.70    | 0.50-0.85       | 压力修正下限；低于 0.50 时大地图对难度的减益过于明显                      |
| `pressure_modifier_max` | float   | 1.50    | 1.20-2.00       | 压力修正上限；高于 2.00 时小地图难度将与主流程数值配置严重失调             |
| `ref_area`              | int     | 2040    | 固定（基准值）   | 压力修正计算的参考面积，更换基准地图时才需调整；MVP 不修改                 |
| `ref_walkable_ratio`    | float   | 0.70    | 固定（基准值）   | 压力修正计算的参考可行走比率，同上；MVP 不修改                            |

**调参原则**：
- 如果 playtest 反馈"地图太空旷，战斗压迫感不足"，优先增加障碍物（降低 `walkable_ratio`），而非缩小地图尺寸。
- 如果刷怪点 < 6，应确保敌人路径仍能形成包围感，否则游戏退化为单向守关。
- 如果塔位 > 7，应同时提高放塔的金币成本，防止玩家早期铺满塔位跳过走位阶段。
- 不要在 MVP 阶段调整 `ref_area` 和 `ref_walkable_ratio`，这两个值在添加第二张地图时才需重新讨论。

**极端值说明**：
- `MAP_WIDTH_PX < 1280` 且 `MAP_HEIGHT_PX < 768` 组合会使压力修正触顶（1.50），同时场面拥挤，不推荐作为正式地图。
- `walkable_ratio > 0.85` 意味着几乎无障碍物，地图将失去空间策略感，仅适合教学场景。

## Open Questions

- 地图障碍物是否应该有被敌人摧毁的机制（v1.0 扩展）？当前 MVP 假设障碍物为永久固定，不需要生命值。
- 第 10 波 Boss 波是否需要特殊的刷怪点（如从地图中心某固定位置出现）？待波次系统设计后确认；当前地图系统预留 8 个刷怪点中可指定 1 个为"里程碑专属刷怪点"。
- 猫粮仓库地图塔位坐标为近似值，需要在 Godot 场景编辑器中实际摆放后确认最终像素坐标，当前数值用于设计评审和接口验证。
- v1.0 引入多张地图时，`ref_area` 和 `ref_walkable_ratio` 的基准值应选择哪张地图作为参考？建议届时以平均值或"最平衡的地图"作为基准。
