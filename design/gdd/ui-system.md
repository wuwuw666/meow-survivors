# UI 系统 (UI System)

> **Status**: In Design
> **Author**: [user + agents]
> **Last Updated**: 2026-04-03
> **Implements Pillar**: 可爱即正义（UI 风格）+ 成长的爽感（数值反馈可视化）+ 策略有深度（信息清晰支持决策）

## Overview

UI 系统（UI System）管理游戏中所有**玩家-facing 的用户界面**，包括：核心 HUD（血条、经验条、金币数、波次信息）、升级选择面板、塔类型选择面板、塔信息面板、游戏暂停菜单、死亡动画期间的过渡 UI。UI 是玩家获取一切游戏信息的唯一通道——必须清晰、及时、可爱。

MVP 覆盖 4 个核心 UI 层级：
1. **HUD 层** — 常驻，显示核心状态（HP/金币/经验/波次/塔位）
2. **面板层** — 临时弹出（升级选择/塔选择/塔信息）
3. **反馈层** — 瞬时飘字（伤害数字/金币拾取/经验拾取）
4. **覆盖层** — 全屏遮罩（暂停菜单/结算画面——由结算系统驱动）

**核心职责**: 数据展示 → 交互面板 → 动画/特效反馈 → 信号订阅

**核心接口**:
- `update_hp(current: int, max: int) -> void`
- `update_coins(amount: int) -> void`
- `update_xp(current: int, needed: int) -> void`
- `update_wave(current: int, total: int) -> void`
- `show_upgrade_panel(options: Array[Dictionary]) -> void`
- `show_tower_selection_panel() -> void`
- `damage_number(position: Vector2, value: int, is_crit: bool, color: String) -> void`

## Player Fantasy

**情感目标：信息一目了然 + 反馈可爱且清晰**

UI 是游戏的"脸面"——可爱风格的字体、圆润的圆角面板、温暖的配色，让玩家从第一眼就觉得"这是一只猫在守护我的仓库"。但同时 UI 必须实用——玩家需要即时看到自己的血量、还剩多少经验升级、金币够不够放塔、现在是第几波。

**玩家应该感受到**：
- "一眼就能看到自己多少血、多少经验、多少金币"
- "升级选择面板弹出来好可爱——字体、图标、背景都暖暖的"
- "伤害数字从敌人头顶飘出来——红的普攻、黄的暴击"
- "金币飞出来'叮'一声数字就跳上去了——爽"

**玩家不应该感受到**：
- "我的血条在哪里？找不到"（HUD 布局混乱）
- "升级了但没感觉"（缺少面板或动画反馈）
- "这个按钮是干嘛的？"（标签不清）
- "飘字太多了把画面挡住了"（反馈过度）

**参考**：吸血鬼幸存者（极简 HUD、信息清晰）、Plant vs. Zombies（可爱卡通 UI 风格）

## Detailed Rules

### 规则 1：UI 层级架构

```
根 Control (CanvasLayer)
├── HUD_Layer (CanvasLayer Z=1)        # 常驻 HUD
│   ├── HPBar (TextureProgressBar)     # 左上角：玩家血条
│   ├── XPBar (TextureProgressBar)     # 左侧中：经验条
│   ├── CoinLabel (HBoxContainer)      # 左下角：金币数
│   ├── WaveLabel (Label)              # 顶部居中：波次信息
│   └── TowerIndicator (HBoxContainer) # 顶部右侧：塔位占用指示
│
├── PANEL_Layer (CanvasLayer Z=10)     # 临时面板
│   ├── UpgradePanel (PanelContainer)  # 升级选择面板
│   ├── TowerSelectPanel (PanelContainer) # 塔类型选择面板
│   └── TowerInfoPanel (PanelContainer)   # 塔信息面板
│
├── FEEDBACK_Layer (CanvasLayer Z=5)   # 瞬时反馈
│   └── DamageNumberContainer (Node)   # 伤害/金币/经验飘字
│
└── OVERLAY_Layer (CanvasLayer Z=100)  # 全屏覆盖层
    ├── PauseMenu (Control)             # 暂停菜单
    └── SettlementOverlay (Control)     # 结算画面 (由结算系统驱动)
```

### 规则 2：HUD — 血条 (HP Bar)

```
左上角锚定
尺寸: 200×24px
组成:
    ❤️ 心形图标
    TextureProgressBar (背景=深红条, 前景=粉红条)
    当前HP/最大HP文字 (如"85/100")

颜色:
    HP > 50%: 粉红 (#FF6B9D)
    HP 25%-50%: 橙色 (#FFA500)
    HP < 25%: 红色 (#FF3333) + 轻微脉冲动画

更新:
    监听生命值系统的 HP 变化信号 → 平滑过渡(0.2s Tween)
    player_died 信号 → 血条清零 + 变灰
```

### 规则 3：HUD — 经验条 (XP Bar)

```
左侧中部，血条下方，间隔 8px
尺寸: 150×16px（比血条小）
组成:
    ⭐ 星形图标 + TextureProgressBar + 当前等级/升级需求
    升级时: 满格动画 + "LEVEL UP!" 文字闪现

颜色:
    进度条背景: 深蓝(#1a1a2e)
    进度条前景: 蓝紫渐变(#667eea → #764ba2)

更新:
    监听经验系统的 XP 变化信号
    升级时: 条满 → 闪烁 0.5s → 清空 → 进入下一等级
    升级面板弹出时: 经验条暂停更新(面板关闭后恢复)
```

### 规则 4：HUD — 金币显示

```
左下角锚定
组成:
    🐟 金币图标 + 当前金币数量 (如 "💰 25")

更新:
    监听金币系统的金币变化信号
    金币变化时: 数字跳动动画(scale 1.0 → 1.15 → 1.0, 0.15s)
    +金币时: 飘出绿色 "+X" (2s 后消失)
    -金币时: 飘出红色 "-X" (1s 后消失)
```

### 规则 5：HUD — 波次信息

```
顶部居中锚定
组成:
    "第 X 波 / 共 Y 波" (如 "第 3 波 / 共 10 波")
    Boss 波时: 添加 "⚠️ BOSS 来袭！" 文字和红色闪烁背景

状态变化:
    波次进行中: 白色文字，"第 X 波"
    波次结束/间歇期: 黄色文字，"第 X 波完成！准备中..."
    Boss 波: 红色脉冲文字，"第 10 波 — BOSS 来袭！"

更新:
    监听波次系统的波次变化信号
```

### 规则 6：HUD — 塔位指示

```
顶部右侧锚定
组成:
    一系列小图标表示每个塔位
    空塔位: 灰色底座图标
    有塔: 对应塔类型图标 (鱼干/毛线/猫薄荷)
    满级塔: 图标右下角显示 ★★★ (等级)

点击交互:
    空塔位 → 触发塔位放置系统的 slot 交互
    有塔 → 弹出塔信息面板
```

### 规则 7：升级选择面板

```
屏幕居中弹出 (模态遮罩 60% 半透明黑)
组成:
    标题: "选择一个升级！"
    3-4 个选项卡, 每个包含:
        - 图标 (大, 64×64px)
        - 名称 (16px 粗体)
        - 描述 (12px)
        - 边框 (选中时高亮)

    [选项1]  [选项2]  [选项3]
    ┌─────┐  ┌─────┐  ┌─────┐
    │ 🐟  │  │ ❤️  │  │ 🎯  │
    │鱼干  │  │ 生命 │  │ 射程 │
    │伤害  │  │+20% │  │+30px│
    │+5    │  │     │  │     │
    └─────┘  └─────┘  └─────┘

交互:
    键盘/手柄: 左/右切换, 回车/空格确认
    鼠标: 点击选项确认
    ESC: 不关闭(MVP 不允许跳过升级)
    超时自动选择: 30 秒不选 → 选第一个

打开:
    经验系统发出 levelup 信号 → 打开升级面板
    游戏世界暂停 (process_mode = PAUSED)

关闭:
    玩家选择后:
        应用升级效果
        关闭面板
        恢复游戏
        发出 upgrade_selected(upgrade_id) 信号
```

### 规则 8：伤害飘字

```
FEEDBACK_Layer 上以世界坐标定位
出现位置: 敌人头顶 (跟随 0.5s 后停止跟随)

样式:
    普通伤害: 红色文字, 从下往上飘(1s), fade out
    暴击伤害: 黄色大文字 + 白色描边 + 星星粒子, 1.2s fade out
    治疗: 绿色文字 "+X", 256px 从下往上飘, 1s fade out
    金币拾取: 金色 "+X 💰", 从金币位置飘向金币 HUD 方向
    经验拾取: 紫色 "+XP", 类似金币

约束:
    同屏同位置最多显示 3 个飘字 (超过的排队或省略)
    高频命中 (自动攻击攻速快时) 做合并: 0.1s 内的伤害合并显示
```

### 规则 9：UI 风格

```
面板风格: 圆角矩形 (corner_radius = 12px)
背景色: #2d2d3d (深蓝灰)
边框色: #4a4a6a (浅蓝灰)
标题文字: #ffffff (白色), 16px 粗体
正文文字: #c0c0d0 (浅灰), 12-14px
强调色: #FFB74D (暖橙) — 按钮、高亮、重要信息
禁用色: 50% 透明度的灰色

字体:
    MVP: Godot 内置系统字体
    可爱主题: 圆角字体 (如 Rounded M+), 如果打包允许

图标风格:
    2D 像素/卡通风格, 32×32px 基础大小
    所有图标圆角/圆形, 不锐利
    符合"可爱即正义"支柱
```

## Formulas

### 公式 1：金币飘字动画

```
飘字初始位置 = 金币掉落物的世界坐标
飘字结束位置 = HUD 金币标签的屏幕坐标
动画:
    前 0.5s: 从世界坐标飞到屏幕坐标
    后 0.5s: 在 HUD 标签位置缩放弹跳 (scale 1.0 → 1.2 → 1.0)
总时长: 1.0s
```

### 公式 2：血条颜色插值

```
hp_ratio = current_hp / max_hp

if hp_ratio > 0.5:
    color = lerp(粉红, 橙色, (1.0 - hp_ratio) × 2.0)
elif hp_ratio > 0.25:
    color = lerp(橙色, 红色, (0.5 - hp_ratio) × 4.0)
else:
    color = 红色

脉冲:
    if hp_ratio < 0.25:
        anim_pulse(scale: 1.0 ↔ 1.02, freq: 3Hz)
```

### 公式 3：伤害数字缩放 (暴击)

```
暴击数字 base_scale = 1.5
出现时:
    scale: 0.0 → 2.0 (弹跳) → 1.5 (0.15s ease_out_back)
    旋转: 0° → 随机(-5° ~ +5°) (增加动感)
普通数字 base_scale = 1.0
出现时:
    scale: 0.8 → 1.0 (0.1s ease_out_quad)
```

### 公式 4：变量汇总

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `hp_bar_width` | int | 200 | 血条宽度 px |
| `xp_bar_width` | int | 150 | 经验条宽度 px |
| `upgrade_panel_timeout` | float | 30.0 | 升级面板自动选择超时 |
| `damage_number_max_per_pos` | int | 3 | 同位置最大飘字数 |
| `damage_merge_window` | float | 0.1 | 伤害合并时间窗 |
| `panel_corner_radius` | int | 12 | 面板圆角半径 |
| `overlay_bg_alpha` | float | 0.6 | 模态遮罩透明度 |

## Edge Cases

| 编号 | 边界情况 | 处理方式 |
|------|---------|---------|
| EC-01 | 升级面板打开时玩家死亡 | 升级面板关闭 → 结算系统接管 → 显示结算画面 |
| EC-02 | 多个升级信号同时发出 (不应该发生, 但防御) | 只显示第一个升级面板, 其他的加入队列等待 |
| EC-03 | 同屏 50+ 伤害飘字 | 超过上限的飘字被省略或合并;"低优先级"飘字 (小伤害) 优先省略 |
| EC-04 | 窗口尺寸变化 | Godot CanvasLayer 自动缩放; 面板锚点确保布局正确 |
| EC-05 | 伤害数字出现在屏幕外 (敌人在视口边缘) | 跟随摄像机视口裁剪: 屏幕外的飘字仍显示但不跟随世界坐标 |
| EC-06 | 经验条满但升级面板延迟 | 经验条保持在满格状态直到面板关闭 |
| EC-07 | Boss 血条 (大尺寸) 与普通 HUD 冲突 | Boss 存活时, HUD 血条位置上移 20px 为 Boss 血条让出空间 |
| EC-08 | 暂停菜单覆盖升级面板 | 暂停菜单 Z=10 覆盖升级面板 Z=10 (同层级后打开的在上面) |

## Dependencies

### 上游依赖

| 系统 | 依赖类型 | 接口 | 说明 |
|------|---------|------|------|
| **生命值系统** | 硬依赖 | `player_hp_changed(current, max)`, `player_died` | 更新血条, 处理死亡 |
| **经验系统** | 硬依赖 | `xp_changed(current, needed)`, `level_up` | 更新经验条, 弹出升级面板 |
| **金币系统** | 硬依赖 | `coins_changed(amount, reason)` | 更新金币数字 + 飘字 |
| **波次系统** | 硬依赖 | `wave_changed(current, total)`, `boss_wave` | 更新次信息 |
| **自动攻击系统** | 软依赖 | `projectile_hit_enemy(enemy, damage, is_crit)` | 伤害飘字 |
| **防御塔系统** | 软依赖 | `tower_placed`, `tower_projectile_hit` | 塔位指示 + 塔伤害飘字 |
| **升级选择系统** | 软依赖 | 提供升级选项数据 | 升级面板内容 |

### 下游依赖

| 系统 | 依赖类型 | 接口 | 说明 |
|------|---------|------|------|
| **结算系统** | 硬依赖 | `show_settlement(stats: Dictionary)` | 结算系统调用此接口展示结算画面 |
| **升级选择系统** | 软依赖 | 玩家点击升级选项 | UI 将选择回传给升级系统 |
| **塔位放置系统** | 软依赖 | 玩家点击塔位 | UI 传递点击给塔位放置系统 |

## Tuning Knobs

| 参数 | 默认值 | 范围 | 影响 |
|------|--------|------|------|
| `damage_merge_window` | 0.1s | 0.05-0.3s | 太短飘字太多; 太长看不到每次命中反馈 |
| `damage_number_max_per_pos` | 3 | 1-8 | 太多遮挡画面; 太少合并过度丢失信息 |
| `upgrade_panel_timeout` | 30s | 20-60s | 太短玩家来不及看选项; 太长等待无聊 |
| `hp_bar_pulse_threshold` | 25% | 15%-40% | 脉冲提示"快死了"; 太高频繁闪烁引起恐慌 |
| `overlay_bg_alpha` | 0.6 | 0.3-0.8 | 太低背景暴露干扰; 太高面板外太黑 |
