# Game Concept: 喵族幸存者 (Meow Survivors)

*Created: 2026-04-01*  
*Status: Under Review*  
*Last Updated: 2026-04-09*

---

## Elevator Pitch

> 这是一款以塔防为主导的可爱猫咪波次生存游戏。你控制一只猫咪英雄在战场上移动补位、收集资源并提供自动攻击支援，同时围绕固定塔位建立防线，抵挡一波又一波越来越强的敌人。你通过经验升级和塔位经营逐步完善自己的塔阵与战斗风格，体验从“勉强守住”到“整条防线成型”的成长快感。

**10秒测试**：你是一只守仓库的猫咪指挥官，边跑边补位，边靠塔阵挡住敌潮，在短局波次里快速成长。

## Core Fantasy

**你是一只在前线补位、但真正依靠塔阵守住仓库的猫咪指挥官。**

你不是单枪匹马清屏的传统幸存者主角，而是防线的调度者和补位者。你要在战场上移动、拉扯、拾取资源、做经验升级决策，同时依靠固定塔位构建一套越来越完整的防线。真正的满足感来自：

- “这座塔放对了。”
- “这个升级拿完，整条线一下稳了。”
- “我不是在硬撑，我是在让防线滚雪球。”

---

## Unique Hook

**塔防主导 + 幸存者移动支援的混合体验**

大多数幸存者游戏让角色本体成为绝对主角，而《喵族幸存者》把重心转向**塔阵构筑**。玩家角色依然重要，但主要承担移动补位、资源运营和局部支援。真正决定一局手感和策略深度的，是你如何围绕固定塔位建立防线，以及你如何让经验成长和塔阵联动起来。

> “像短局幸存者一样快速成长，但真正的主角是你搭出来的猫咪防线。”

## Player Experience Analysis (MDA Framework)

### Target Aesthetics（目标美学）

| 美学 | 优先级 | 如何实现 |
| ---- | ---- | ---- |
| **Sensation** (感官愉悦) | 高 | 可爱视觉、清晰打击反馈、金币/经验获取反馈 |
| **Fantasy** (幻想) | 中高 | 扮演可爱猫咪守护仓库、指挥防线成长 |
| **Narrative** (叙事) | 低 | 保持轻量背景设定，服务玩法即可 |
| **Challenge** (挑战) | 高 | 波次节奏、塔位取舍、升级路线选择 |
| **Discovery** (探索) | 中 | 尝试不同塔与升级联动，找出高效组合 |
| **Expression** (表达) | 高 | 不同防线布局、不同升级路线形成不同局内风格 |
| **Submission** (放松) | 中 | 短局、易重开、操作简单但决策有层次 |

### Key Dynamics（涌现行为）

- 玩家会思考“这一波我该先补塔还是先拿角色成长”。
- 玩家会反复尝试更优的塔位顺序和升级顺序。
- 玩家会因为防线成型而获得明显的安全感与掌控感。
- 玩家失败后会复盘“如果关键塔再早一点成型，这局就过了”。

### Core Mechanics（核心机制）

1. **固定塔位布防**：地图上存在明确塔位，玩家围绕这些位置建立核心防线。
2. **移动补位 + 自动攻击支援**：玩家控制猫咪移动、拉扯和补输出，但不是唯一主力。
3. **波次系统**：敌人按波次进攻，节奏明确但间隔偏短。
4. **经验升级**：主要成长入口来自经验升级，每次从 3-4 个选项中选择一个。
5. **塔与角色联动成长**：升级既能强化角色，也能强化塔阵和塔间联动。

## Player Motivation Profile

### Primary Psychological Needs（主要心理需求）

| 需求 | 如何满足 | 强度 |
| ---- | ---- | ---- |
| **Autonomy（自主性）** | 塔位选择、升级路线选择、补位路径选择 | 核心 |
| **Competence（能力感）** | 从守不住到守得住，再到防线成型碾压 | 核心 |
| **Relatedness（关联感）** | 猫咪主题、角色与塔的陪伴感 | 支持 |

### Player Type Appeal（玩家类型）

- **Achievers（成就者）**：追求更高波次、更稳的通关、更高效的 build。
- **Explorers（探索者）**：尝试不同塔阵组合、升级联动和站位策略。
- **Socializers（社交者）**：不是主目标玩家。
- **Killers / Competitors（竞争者）**：不是 MVP 的优先目标。

### Flow State Design（心流设计）

- **新手引导曲线**：前几波快速理解“补位、建塔、升级”三个动作。
- **难度递增**：敌人每波更强，但玩家也通过经验升级和塔阵经营变强。
- **反馈清晰**：玩家要能看懂哪条防线变强了、哪次升级生效了。
- **失败恢复**：短局和快速重开降低挫败感。

## Core Loop

### 30秒循环（瞬间体验）

```text
观察敌人推进路线
    ↓
围绕塔位移动补位
    ↓
塔阵与角色一起清怪
    ↓
敌人死亡 → 金币/经验飞出
    ↓
拾取资源并调整站位
    ↓
（循环）
```

**核心动作**：围绕防线做位置决策。玩家不是单独找输出位，而是在“哪里补位最能让这条线稳住、哪里最能吃到经验和资源”之间做取舍。

**内在满足感**：防线逐渐稳定、清怪效率上升、经验与塔阵同时变强的滚雪球快感。

### 5分钟循环（短期目标）

```text
进入一波敌人
    ↓
战斗（约20-45秒）
    ↓
拾取经验并触发升级
    ↓
选择升级（3-4选1）
    ↓
角色/塔阵联动变强
    ↓
短暂整理后进入下一波（敌人更强）
    ↓
（循环）
```

**玩家决策**：

- 强化角色支援能力 vs 强化塔输出 vs 强化防线稳定性
- 哪个塔位先成型
- 当前经验升级是服务这一波，还是为后续波次做铺垫

### 会话循环（一局游戏）

```text
选择猫咪英雄
    ↓
第1波 → 第2波 → ... → 第10波
    ↓
Boss波（高潮挑战）
    ↓
死亡/通关 → 结算展示
    ↓
“再来一局”（尝试新塔阵 / 新升级路线）
```

**自然停止点**：死亡后的结算画面。  
**回归钩子**：“这次如果早点把那座关键塔做出来，应该能稳过Boss。”

### Long-Term Progression（长期成长）

以下内容保留为长期方向，但**不作为当前 MVP 的设计重心**：

- 解锁新猫咪英雄
- 解锁新防御塔类型
- 轻量永久成长
- 成就系统

## Game Pillars

### Pillar 1: 可爱即正义

所有视觉元素都必须可爱、温暖、让人想继续看下去，不做阴暗、血腥、压抑的表达。

### Pillar 2: 成长的爽感

每次升级都要让玩家明显感受到“我的角色更有用了”或“我的防线更成型了”。

### Pillar 3: 策略有深度

塔位、升级、补位路线都必须形成真实权衡，不允许所有最优解永远固定。

### Anti-Pillars（不做的事）

- 不做复杂连招或高精度操作
- 不做黑暗恐怖或血腥风格
- 不做多人 / PVP
- 不在 MVP 阶段做重度 meta progression

## Risks and Open Questions

### Design Risks

- 塔防主导和角色支援的权重可能失衡
- 经验升级如果只强化角色，可能会削弱塔阵主角感
- 塔位数量过少会影响策略深度，过多会稀释短局节奏

### Technical Risks

- 多塔、多敌人、多投射物同屏下的性能压力
- 塔阵联动和 buff 系统如果过早复杂化，会拖慢 MVP

### Scope Risks

- 如果过早追求多塔、多英雄、多地图，单人开发范围会迅速失控
- 如果在 MVP 里加入重 meta，会模糊核心循环验证结果

### Open Questions

- 每张地图更适合 3 个、4 个还是 5 个关键塔位？
- 经验升级里有多少比例应该直接服务塔，而不是服务角色？
- 输出塔和辅助塔的比例怎样设计，才能保证既有策略感又不拖节奏？

## MVP Definition

**核心假设**：玩家会觉得“固定塔位布防 + 移动补位支援 + 经验升级成长 + 短局波次推进”的循环有趣，并愿意连续玩多局。

**MVP必须包含**：

1. 1只可玩的猫咪英雄
2. 2类核心塔定位（输出型、辅助型）
3. 控制效果优先作为输出塔的分支效果或升级效果存在，而不是独立主塔类型
4. 5种敌人（小怪3种 + 精英1种 + Boss 1种）
5. 10个升级选项
6. 1张地图（猫粮仓库主题）
7. 10波敌人
8. 基本UI（血条、经验条、金币、波次、升级面板）
9. 简单结算画面

**MVP明确不做**：

- 多猫咪英雄
- 重度解锁系统 / 重度 meta progression
- 多张地图
- 大量塔类型扩张
- 音效/音乐完整包装

### Scope Tiers

| 阶段 | 内容 | 功能 | 时间线 |
| ---- | ---- | ---- | ---- |
| **MVP** | 1英雄 + 2类核心塔 + 5敌人 + 10升级 + 1地图 + 10波 | 核心循环验证 | 2-4周 |
| **v1.0** | 多英雄 + 更多塔类型 + 更多敌人 + 多地图 + 无尽模式 | 正式发布版本 | 3-6个月 |
| **完整愿景** | 扩展内容池 + 轻量 meta + 长线更新 | 长期更新 | 6-12个月 |
